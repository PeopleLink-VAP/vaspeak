## SpacetimeDB Tables

SpacetimeDB uses Rust-based table definitions. Below are the planned schemas:

### `lessons` Table
```rust
#[table(name = lessons, public)]
pub struct Lesson {
    #[primary_key]
    pub id: Identity,
    pub day_number: u32,
    pub week_theme: String,
    pub title: String,
    pub content: String, // JSON serialized block structure
    pub created_at: i64,
    pub updated_at: i64,
}
```

**JSONB Content Structure:**
```json
{
  "blocks": [
    {
      "type": "listening_decoding",
      "audio_url": "...",
      "transcript": "...",
      "questions": [...]
    },
    {
      "type": "pattern_drilling",
      "phrases": [...],
      "repetitions": 3
    },
    {
      "type": "guided_simulation",
      "difficulty": "practice|real|stress",
      "scenario": "...",
      "prompts": [...]
    },
    {
      "type": "emotional_reflection",
      "prompts": [...],
      "self_assessment_scale": 5
    }
  ]
}
```

### `profiles` Table
```rust
#[table(name = profiles, public)]
pub struct Profile {
    #[primary_key]
    pub id: Identity,
    pub email: String,
    pub display_name: Option<String>,
    pub current_level: String, // "survival_speaker", "working_va", etc.
    pub native_language: String,
    pub created_at: i64,
}
```

### `user_progress` Table
```rust
#[table(name = user_progress, public)]
pub struct UserProgress {
    #[primary_key]
    pub id: Identity,
    pub user_id: Identity,
    pub lesson_id: Identity,
    pub block_completions: String, // JSON serialized
    pub simulation_scores: String, // JSON serialized
    pub stress_level: Option<u32>,
    pub reflection_notes: Option<String>,
    pub completed_at: Option<i64>,
}
```

## Access Control

SpacetimeDB uses identity-based access:
- Users authenticate via SpacetimeDB Identity
- Row-level filtering handled in reducer functions
- `lessons` are readable by all authenticated users
- Users can only read/write their own `profiles` and `user_progress`

## Deployment Pipeline

### Netlify Configuration
```toml
# netlify.toml
[build]
  command = "npm run build"
  publish = "build"

[build.environment]
  NODE_VERSION = "22"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### Environment Variables
```bash
# .env.example
SPACETIMEDB_URI=your_spacetime_uri
SPACETIMEDB_MODULE=your_module_name
RESEND_API_KEY=your_resend_key
GROQ_API_KEY=your_groq_key
GROQ_MODEL=meta-llama/llama-4-scout-17b-16e-instruct
```

### Build & Deploy Flow
1. Push to `main` branch
2. Netlify triggers build
3. `npm run build` runs (SvelteKit → static + functions)
4. Deploy to Netlify CDN
5. Edge functions deployed globally

## API Endpoints (Planned)

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/auth/callback` | GET | SpacetimeDB auth callback |
| `/api/lessons` | GET | Fetch all lessons |
| `/api/lessons/:id` | GET | Fetch single lesson |
| `/api/progress` | GET/POST | Read/write user progress |
| `/api/profile` | GET/PATCH | User profile management |
| `/api/placement` | POST | Submit placement test |

## Security Considerations

1. **Authentication**: SpacetimeDB Identity
2. **Authorization**: Identity-based row filtering in reducers
3. **WebSocket**: Encrypted WSS connections to SpacetimeDB
4. **CORS**: Configured via Netlify headers
5. **Rate Limiting**: Consider for API endpoints (future)
