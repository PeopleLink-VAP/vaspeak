## Database Schema

### `lessons` Table
```sql
CREATE TABLE lessons (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  day_number INT UNIQUE NOT NULL,
  week_theme TEXT NOT NULL,
  title TEXT NOT NULL,
  content JSONB NOT NULL,  -- Flexible block structure
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**JSONB Content Structure:**
```json
{
  "blocks": [
    {
      "type": "listening_decoding",
      "audio_url": "...",
      "transcript": "...",
      "questions": [...]
    },
    {
      "type": "pattern_drilling",
      "phrases": [...],
      "repetitions": 3
    },
    {
      "type": "guided_simulation",
      "difficulty": "practice|real|stress",
      "scenario": "...",
      "prompts": [...]
    },
    {
      "type": "emotional_reflection",
      "prompts": [...],
      "self_assessment_scale": 5
    }
  ]
}
```

### `profiles` Table
```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  email TEXT,
  display_name TEXT,
  current_level TEXT DEFAULT 'survival_speaker',
  native_language TEXT DEFAULT 'vi',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### `user_progress` Table
```sql
CREATE TABLE user_progress (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) NOT NULL,
  lesson_id UUID REFERENCES lessons(id) NOT NULL,
  block_completions JSONB DEFAULT '{}',
  simulation_scores JSONB DEFAULT '{}',
  stress_level INT,
  reflection_notes TEXT,
  completed_at TIMESTAMPTZ,
  UNIQUE(user_id, lesson_id)
);
```

## Row-Level Security (RLS) Policies

All tables must implement RLS to ensure:
- Users can only read/write their own `profiles` and `user_progress`
- `lessons` are readable by all authenticated users
- No unauthorized data access

## Deployment Pipeline

### Netlify Configuration
```toml
# netlify.toml
[build]
  command = "npm run build"
  publish = "build"

[build.environment]
  NODE_VERSION = "22"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### Environment Variables
```bash
# .env.example
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_key  # Server-side only
RESEND_API_KEY=your_resend_key  # Future
```

### Build & Deploy Flow
1. Push to `main` branch
2. Netlify triggers build
3. `npm run build` runs (SvelteKit → static + functions)
4. Deploy to Netlify CDN
5. Edge functions deployed globally

## Security Considerations

1. **Authentication**: Supabase GoTrue with email/password
2. **Authorization**: RLS policies on all tables
3. **API Keys**: Service role key never exposed to client
4. **CORS**: Configured via Netlify headers
5. **Rate Limiting**: Consider for API endpoints (future)
