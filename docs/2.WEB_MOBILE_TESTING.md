# VASpeak - Web & Mobile Testing

## Testing Philosophy

Given the target audience (Vietnamese VAs accessing via mobile devices), testing must prioritize:
1. **Mobile-first responsive design**
2. **Audio recording/playback reliability**
3. **Cross-browser compatibility**
4. **Performance on low-end devices**

## Test Categories

### 1. Smoke Tests (Critical Path)
Quick validation that core flows work after deployment.

```typescript
// tests/smoke.test.ts
- Homepage loads
- Navigation works
- Auth pages render
- Protected routes redirect when unauthenticated
```

### 2. Authentication Tests
```typescript
// tests/auth.test.ts
- User can sign up
- User can log in
- User can log out
- Protected routes accessible after login
- Session persists across page reload
- RLS policies enforced
```

### 3. Lesson Flow Tests
```typescript
// tests/lesson.test.ts
- Lessons list displays correctly
- Individual lesson loads with all blocks
- Block completion updates progress
- Progress persists after completion
- Navigation between lesson days works
```

### 4. Audio Recording Tests
```typescript
// tests/audio.test.ts
- Microphone permission request
- Recording starts/stops
- Audio playback works
- Recording upload succeeds
```

### 5. Mobile Responsiveness Tests
```typescript
// tests/mobile.test.ts
- Layout adapts to mobile viewport
- Touch interactions work
- Audio controls are accessible
- Text is readable on small screens
- Navigation is thumb-friendly
```

## Test Configuration

### Playwright Config
```typescript
// playwright.config.ts
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  reporters: ['html'],
  use: {
    baseURL: 'http://localhost:5173',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
    {
      name: 'Mobile Chrome',
      use: { ...devices['Pixel 5'] },
    },
    {
      name: 'Mobile Safari',
      use: { ...devices['iPhone 12'] },
    },
  ],
  webServer: {
    command: 'npm run dev',
    url: 'http://localhost:5173',
    reuseExistingServer: !process.env.CI,
  },
});
```

## Mobile Testing Strategy

### Target Devices (Vietnam Market)
| Device | Viewport | Priority |
|--------|----------|----------|
| iPhone 12/13 | 390 x 844 | High |
| Samsung Galaxy A series | 360 x 800 | High |
| iPhone SE | 375 x 667 | Medium |
| iPad | 768 x 1024 | Low |

### Mobile-Specific Tests

#### Touch Interactions
- Tap targets minimum 44x44px
- Swipe gestures for lesson navigation
- Pull-to-refresh (if implemented)
- Long-press for additional options

#### Audio on Mobile
- Autoplay policies (user gesture required)
- Audio works with screen locked (PWA)
- Bluetooth headphone compatibility
- Speaker/headphone switching

#### Performance on Low-End Devices
- Initial load under 3 seconds on 3G
- Time to interactive under 5 seconds
- Memory usage under 100MB
- Battery impact minimal

## Running Tests

### Local Development
```bash
# Run all tests
npx playwright test

# Run specific test file
npx playwright test tests/smoke.test.ts

# Run with UI mode (debugging)
npx playwright test --ui

# Run on specific browser
npx playwright test --project=chromium

# Run mobile tests only
npx playwright test --project="Mobile Chrome"
```

### CI/CD Integration
```bash
# Install Playwright browsers
npx playwright install --with-deps

# Run tests in CI mode
npx playwright test --reporter=html
```

## Performance Testing

### Core Web Vitals Targets

|--------|--------|----------|
